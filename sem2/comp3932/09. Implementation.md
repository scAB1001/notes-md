## Setting Up The HDRP Unity Scene
Here is the structured approach to building your foundation, combining both Varjo's visual requirements and Ultraleap's kinematics.
### 1. HDRP Configuration
To ensure the Varjo headset renders the environment correctly without visual artifacts, configure your pipeline settings as follows:
- **Color Buffer:** Navigate to the Project Settings dialog > Quality tab > HDRP settings. Ensure the **Color Buffer Format** is set strictly to `R16G16B16A16` to support the alpha channel needed for mixed reality (Found in **Theory pt.2**, under **High Definition Render Pipeline and Universal Render Pipeline**).
- **Post-Processing:** Create a global volume profile in your scene. You must apply specific overrides to prevent foveation artifacts (Found in **Theory pt.1**, under **HDRP Post Process Settings**):
    - **Exposure:** Mode = Fixed (adjust value based on your scene lighting).
    - **Bloom:** Intensity = 0.
    - **Ambient Occlusion:** Intensity = 0.
- **Camera Clear:** To ensure Video Pass-through (VPT) works smoothly, select your Main Camera, set the Background Type to Color, and set the background to `RGBA(0,0,0,0)`.
### 2. Implementing Hand Tracking (Ultraleap)
Your XR-4 headset uses standard OpenXR hand tracking, but you will use the Ultraleap Unity plugin to drive the interactions.
1. **Rig Setup:** In your Hierarchy, navigate to `GameObject > XR > XR Origin`.
2. **Add Hands:** Right-click the `XR Leap Provider Manager` > `Ultraleap` > `Hands` > `Capsule Hands`.
3. **Offset Configuration:** Because the XR-4 aligns the head and hand tracking origins, you must ensure the offset is zeroed out. Attach the custom `VarjoHandTrackingOffset` script (detailed in **Theory pt.2**, under **Set hand tracking offsets automatically**) to your `LeapXRServiceProvider`. It will automatically set the `deviceOffsetYAxis`, `deviceOffsetZAxis`, and `deviceTiltXAxis` to `0f` when it detects the XR-4.
### 3. Implementing Eye Tracking (Varjo)
With hands rendering, you can initialize the 20/20 Eye Tracker to begin logging gaze data or using it as a secondary interaction method.
1. **Calibration:** Eye tracking must be calibrated. Call `VarjoEyeTracking.RequestGazeCalibration()` at the start of your application.
2. **Data Polling:** Once calibrated, you can poll `VarjoEyeTracking.GetGaze()` in your `Update()` loop to retrieve the `GazeData` struct.
3. **Filtration:** By default, the gaze data uses a "Standard" filter. If you need raw, unfiltered data for your evaluation metrics, use `VarjoEyeTracking.SetGazeOutputFilterType(None)`.

### 1. Tracking Hinting API – High-Fidelity Mode  
**Hint:** `LEAP_HINT_HIGH_HAND_FIDELITY`  

- **What it is:** A performance-mode hint that tells the Ultraleap Tracking Service to **maximize CPU usage** for hand tracking instead of conserving power.  
- **Why it matters:**  
  - You’re on a powerful PC (i7-11700 + RTX 3080), so thermal/battery constraints don’t apply.  
  - You need the **cleanest possible kinematic data** for your `AlignmentMetricsLogger`.  
- **How to use it:**

  - In Unity/C#, this typically means configuring the tracking service via the config API or setting the hint before starting tracking.  
  - Example concept (pseudo-code):
    ```csharp
    // Pseudo-example: set high-fidelity hint before tracking starts

    ```

### 2. Tracking Hinting API – Microgestures Mode  
**Hint:** `LEAP_HINT_MICROGESTURES`  

- **What it is:** A tracking-model hint optimized for **subtle, fine finger movements** (microgestures).  
- **Why it matters:**  
  - Your gesture vocabulary is **very strict**:  
    - Pinch “Enter” threshold: `0.02 m`  
    - Pinch “Exit” threshold: `0.05 m`  
  - If the default model prioritizes gross arm motions, it may **smooth over** or miss the exact millimeter crossing point.  
  - Microgesture mode ensures the tracker **prioritizes fingertip accuracy**.  
- **How to use it:**  
  - Set this hint in your tracking service initialization, ideally **alongside** `HIGH_HAND_FIDELITY`.  
  - Concept:
    ```csharp
    // Pseudo-example: enable microgesture tracking model
    Leap.Service.SetHint(Leap.LeapConstants.LeapHintMicrogestures, "true");
    ```

***
### 3. `CapsuleHand` Color Manipulation  
**Properties:**  
- `Color SphereColour { get; set; }`  
- `Color CylinderColour { get; set; }`  
**Method:** `void SetCustomColours(Color sphere, Color cylinder)`  

- **What it is:** Dynamic runtime control over the **visual colors** of the virtual hand’s spheres and cylinders.  
- **Why it matters:**  
  - You lack physical haptic feedback.  
  - You can **replace haptics with visual feedback**:  
    - When a user’s pinch distance crosses `0.02 m`, instantly change the hand color.  
    - This gives **immediate visual confirmation** of gesture acceptance, reducing mapping dissonance.  
- **How to use it:**  
  - Attach a script to your `CapsuleHand` (or get a reference to it from the `XR Leap Provider Manager`).  
  - In your gesture logic (e.g., in `HandMetricsLogger` or a gesture detector), detect when:
    ```csharp
    if (pinchDistance <= 0.02f) {
        capsuleHand.SetCustomColours(activePinchColor, activePinchColor);
    } else if (pinchDistance > 0.05f) {
        capsuleHand.SetCustomColours(normalColor, normalColor);
    }
    ```
  - Use distinct colors for:
    - Normal state  
    - Pinch-in-progress  
    - Confirmed pinch
